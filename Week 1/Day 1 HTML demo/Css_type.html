CSS Styling Approaches

Inline, internal, and external CSS each solve different problems. This page renders all three so you can spot when each technique shines.

Inline CSS

Inline styles live inside the HTML tag via the style attribute. They override other declarations on the same element.

The following pill uses inline CSS to control its colors and spacing:

Inline style applied! 
<span style="background:#f97316; border:2px solid #ea580c;">...</span>

Great for quick demos or generated markup, but avoid for large projects to keep presentation separate from content.

Internal (Embedded) CSS

Embedded styles appear inside a <style> block within the document’s <head>. Everything you are reading now uses internal CSS for layout, spacing, and colors.

Inspect this page to see selectors such as body, header, and .panel defined directly in the head.

Internal CSS keeps the styles alongside the markup, useful for component prototypes or single-file demos.

External CSS

External stylesheets live in their own files and apply everywhere they are linked. That keeps design consistent across multiple pages.

This paragraph picks up its design from styles.css, proving the link tag at the top is working.

<link rel="stylesheet" href="styles.css">

External CSS is the go-to choice for production websites so you can cache, version, and reuse styling rules.
