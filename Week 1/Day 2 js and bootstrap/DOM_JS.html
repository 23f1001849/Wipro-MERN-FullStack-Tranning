DOM Tree Cheat Sheet

The DOM represents your document as a tree. Each element, attribute, and piece of text is a node that can be queried and modified.

Document
└── html
    ├── head
    │   ├── meta
    │   ├── meta
    │   └── title
    └── body
        ├── header
        ├── main
        │   └── section
        │       ├── h2
        │       └── p
        └── footer

Essential Selection Methods

Use these DOM APIs to find elements in your document. Each method returns different types of collections.

document.getElementById()

Returns the first element with the matching id. Perfect for grabbing unique regions.

document.querySelector()

Accepts any CSS selector and returns the first matching element. Great for concise lookups.

document.querySelectorAll()

Returns a static NodeList of all matches. Loop with forEach or convert to an array.

element.classList

Add, remove, or toggle classes to change styling without touching inline styles.

Common Attribute Helpers

Beyond changing text and classes, you can manage metadata with attribute helpers:

  • element.setAttribute(name, value) — create or update an attribute.
  • element.getAttribute(name) — read the current value.
  • element.removeAttribute(name) — remove the attribute entirely.
